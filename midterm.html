Quizzes

Quiz 1
Introduction to OOP
▾

1. What is Object-Oriented Programming?

Answer

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects" that contain data (fields/attributes) and code (methods). It organizes software design around data rather than functions and logic.

OOP models real-world entities as objects. For example, a Car object has attributes like color and speed, and methods like drive() and brake(). This makes code more modular, reusable, and maintainable.

2. What is the difference between a class and an object?

Answer

A class is a blueprint that defines attributes and behaviors. An object is a specific instance of a class created using the new keyword.

Think of a class as a cookie cutter and an object as the actual cookie. class Dog { } defines what a Dog is, while Dog myDog = new Dog(); creates one in memory.

3. What are the four pillars of OOP?

Answer

Encapsulation, Inheritance, Polymorphism, and Abstraction.

These four principles form the foundation of OOP. Encapsulation bundles data with methods; Inheritance lets classes derive from others; Polymorphism allows multiple forms; Abstraction hides complexity.

4. What is a constructor in Java?

Answer

A constructor is a special method invoked automatically when an object is created. It has the same name as the class and no return type.

Constructors initialize object state. Java provides a default no-arg constructor if none is defined. You can create parameterized constructors: public Student(String name) { this.name = name; }

5. What are access modifiers in Java?

Answer

Access modifiers control visibility: public, private, protected, and default (package-private).

public — accessible everywhere; private — same class only; protected — same package + subclasses; default — same package only. These enforce encapsulation.

Quiz 2
Encapsulation & Methods
▾

1. What is encapsulation and why is it important?

Answer

Encapsulation is bundling data and methods within a class while restricting direct access using access modifiers.

It protects internal state from unintended modification. A BankAccount class hides the balance field and provides deposit() and withdraw() with validation.

2. What is the purpose of getter and setter methods?

Answer

Getters retrieve private field values. Setters allow controlled modification with optional validation logic.

Instead of object.balance directly, we use getBalance() and setBalance(). The setter can include validation like if (amount >= 0).

3. What is method overloading?

Answer

Defining multiple methods with the same name but different parameter lists within the same class. Resolved at compile-time.

add(int a, int b) and add(double a, double b) are overloaded. Java determines which to call based on arguments passed. This is static polymorphism.

4. What is the this keyword in Java?

Answer

this refers to the current object instance. Used to distinguish instance variables from parameters with the same name.

In public Student(String name) { this.name = name; }, this.name is the instance variable while name is the parameter.

Class Design Basics

1. Create a Student class with name, age, and GPA fields. Include a constructor and a display method.

Answer

See code below.

Student.java 
public class Student {
    private String name;
    private int age;
    private double gpa;

    public Student(String name, int age, double gpa) {
        this.name = name;
        this.age = age;
        this.gpa = gpa;
    }

    public void displayInfo() {
        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
        System.out.println("GPA: " + gpa);
    }
}

Demonstrates encapsulation: fields are private and initialized through the constructor. The displayInfo() method provides controlled output.

Seatwork 2
Inheritance Practice
▾

1. Create a Person superclass, then a Student subclass that adds a studentId field.

Answer

See code below.

Person.java 
public class Person {
    protected String name;
    protected int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void display() {
        System.out.println(name + ", Age: " + age);
    }
}

public class Student extends Person {
    private String studentId;

    public Student(String name, int age, String studentId) {
        super(name, age);
        this.studentId = studentId;
    }

    @Override
    public void display() {
        super.display();
        System.out.println("Student ID: " + studentId);
    }
}

Student inherits from Person using extends. The super() call passes arguments to the parent constructor. @Override indicates the method replaces the parent's implementation.

Activity 1
Simple ATM System
▾
ATM.java 
import java.util.Scanner;

public class ATM {
    private double balance;
    private String accountHolder;

    public ATM(String accountHolder, double initialBalance) {
        this.accountHolder = accountHolder;
        this.balance = initialBalance;
    }

    public void checkBalance() {
        System.out.printf("Balance: P%.2f%n", balance);
    }

    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.printf("Deposited: P%.2f%n", amount);
        } else {
            System.out.println("Invalid amount.");
        }
    }

    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.printf("Withdrawn: P%.2f%n", amount);
        } else if (amount > balance) {
            System.out.println("Insufficient funds.");
        } else {
            System.out.println("Invalid amount.");
        }
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        ATM atm = new ATM("Jayvee", 10000.00);
        int choice;

        do {
            System.out.println("\n=== ATM Menu ===");
            System.out.println("1. Check Balance");
            System.out.println("2. Deposit");
            System.out.println("3. Withdraw");
            System.out.println("4. Exit");
            System.out.print("Choose: ");
            choice = sc.nextInt();

            switch (choice) {
                case 1: atm.checkBalance(); break;
                case 2:
                    System.out.print("Amount: ");
                    atm.deposit(sc.nextDouble());
                    break;
                case 3:
                    System.out.print("Amount: ");
                    atm.withdraw(sc.nextDouble());
                    break;
                case 4:
                    System.out.println("Thank you!");
                    break;
                default:
                    System.out.println("Invalid option.");
            }
        } while (choice != 4);
        sc.close();
    }
}
Output
=== ATM Menu === 1. Check Balance 2. Deposit 3. Withdraw 4. Exit Choose: 1 Balance: P10000.00 === ATM Menu === Choose: 2 Amount: 5000 Deposited: P5000.00 === ATM Menu === Choose: 4 Thank you!
Logic
  • Encapsulation — The balance field is private, modified only through deposit() and withdraw() with validation.
  • Constructor — Account holder and balance are initialized at creation.
  • Menu-driven — A do-while loop with switch creates the interactive interface.
Activity 2
Grade Calculator
▾
GradeCalculator.java 
public class GradeCalculator {
    private double grade;
    private String studentName;

    public GradeCalculator(String name, double grade) {
        this.studentName = name;
        this.grade = grade;
    }

    public String getLetterGrade() {
        if (grade >= 90) return "A - Excellent";
        else if (grade >= 80) return "B - Very Good";
        else if (grade >= 70) return "C - Good";
        else if (grade >= 60) return "D - Passed";
        else return "F - Failed";
    }

    public void displayResult() {
        System.out.println("Student: " + studentName);
        System.out.printf("Grade: %.1f%n", grade);
        System.out.println("Letter: " + getLetterGrade());
        System.out.println("Remarks: " + (grade >= 60 ? "PASSED" : "FAILED"));
    }

    public static void main(String[] args) {
        GradeCalculator gc = new GradeCalculator("Jayvee", 85.5);
        gc.displayResult();
    }
}
Output
Student: Jayvee Grade: 85.5 Letter: B - Very Good Remarks: PASSED
Logic
  • If-else ladder — Classifies grades into letter grades.
  • Ternary operator — Concise pass/fail logic.
  • Method decomposition — Each task separated into its own method.
Activity 3
Student Record System
▾
StudentRecords.java 
public class StudentRecords {
    private String[] names;
    private double[] grades;
    private int count;

    public StudentRecords(int capacity) {
        names = new String[capacity];
        grades = new double[capacity];
        count = 0;
    }

    public void addStudent(String name, double grade) {
        if (count < names.length) {
            names[count] = name;
            grades[count] = grade;
            count++;
        }
    }

    public double getAverage() {
        double sum = 0;
        for (int i = 0; i < count; i++) sum += grades[i];
        return count > 0 ? sum / count : 0;
    }

    public String getHighest() {
        int maxIdx = 0;
        for (int i = 1; i < count; i++) {
            if (grades[i] > grades[maxIdx]) maxIdx = i;
        }
        return names[maxIdx] + " (" + grades[maxIdx] + ")";
    }

    public void displayAll() {
        System.out.println("=== Student Records ===");
        for (int i = 0; i < count; i++)
            System.out.printf("%s: %.1f%n", names[i], grades[i]);
        System.out.printf("Average: %.1f%n", getAverage());
        System.out.println("Highest: " + getHighest());
    }

    public static void main(String[] args) {
        StudentRecords r = new StudentRecords(5);
        r.addStudent("Alice", 92.5);
        r.addStudent("Bob", 85.0);
        r.addStudent("Carol", 78.3);
        r.displayAll();
    }
}
Output
=== Student Records === Alice: 92.5 Bob: 85.0 Carol: 78.3 Average: 85.3 Highest: Alice (92.5)
Key Learnings
  • Parallel arrays — names[] and grades[] store data for multiple students.
  • Counter pattern — count tracks entries vs. capacity.
  • Helper methods — getAverage() and getHighest() encapsulate calculations.

OOP Fundamentals
▾

1. Explain the difference between method overloading and method overriding.

Answer

Overloading — same name, different parameters, same class, compile-time. Overriding — same signature, subclass provides new implementation, runtime.

2. Write a program demonstrating inheritance with Shape, Circle, and Rectangle.

Answer

See code below.

ShapeDemo.java 
abstract class Shape {
    abstract double area();

    public void display() {
        System.out.printf("%s area: %.2f%n",
            this.getClass().getSimpleName(), area());
    }
}

class Circle extends Shape {
    private double radius;
    public Circle(double r) { this.radius = r; }

    @Override
    double area() { return Math.PI * radius * radius; }
}

class Rectangle extends Shape {
    private double w, h;
    public Rectangle(double w, double h) { this.w = w; this.h = h; }

    @Override
    double area() { return w * h; }
}

public class ShapeDemo {
    public static void main(String[] args) {
        Shape[] shapes = { new Circle(5.0), new Rectangle(4.0, 6.0) };
        for (Shape s : shapes) s.display();
    }
}
Output
Circle area: 78.54 Rectangle area: 24.00
