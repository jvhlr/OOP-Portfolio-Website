The four pillars of OOP

The foundation of everything in this portfolio.

Encapsulation

Bundling data and methods within a class. Restricting access using access modifiers to protect internal state.

Inheritance

Creating new classes from existing ones. Enables code reuse and establishes "is-a" relationships.

Polymorphism

One interface, many implementations. Objects take different forms through overriding and overloading.

Abstraction

Hiding complexity, exposing only essential features through abstract classes and interfaces.
