Quiz 3
Inheritance & Polymorphism
▾

1. What is the difference between an abstract class and an interface?

Answer

An abstract class can have both abstract and concrete methods, instance variables, and constructors. Supports single inheritance. An interface defines a contract — supports multiple implementation and (since Java 8) default/static methods.

Use abstract classes when classes share common state (e.g., Animal with name). Use interfaces when unrelated classes share a capability (e.g., Printable). "Is-a" vs. "can-do."

2. What is runtime polymorphism?

Answer

The JVM determines which method to call based on the actual object type at runtime, not the reference type. Achieved through method overriding and inheritance.

Shape s = new Circle(); s.area(); — the JVM calls Circle's area() because the actual object is a Circle, even though s is declared as Shape.

3. Can a class implement multiple interfaces?

Answer

Yes. class Dog implements Animal, Pet { }. However, Java does not support multiple class inheritance — only one superclass allowed.

This avoids the "diamond problem." Interfaces allow a class to fulfill multiple contracts without inheriting conflicting implementations.

4. What is the purpose of @Override?

Answer

Tells the compiler the method is intended to override a superclass method. If it doesn't actually override anything, the compiler produces an error.

Without @Override, a typo like tostring() instead of toString() would silently create a new method. The annotation catches this at compile time.

Quiz 4
Abstraction & Interfaces
▾

1. What is abstraction and how does it simplify programming?

Answer

Abstraction hides implementation details and exposes only essential features. It lets developers work with high-level concepts without worrying about internals.

Like driving a car — you interact with the steering wheel (interface), not the engine (implementation). Abstract classes and interfaces define what's available without dictating how.

2. What are default methods in interfaces?

Answer

Methods in an interface with a body. Allow adding new methods without breaking existing implementations. Declared with the default keyword (Java 8+).

Introduced to evolve interfaces over time. Implementing classes automatically get the method but can override it if needed.

Interface Implementation
▾

1. Create a Payable interface. Implement it in FullTimeEmployee and PartTimeEmployee.

PayableDemo.java 
interface Payable {
    double calculatePay();
    String getEmployeeInfo();
}

class FullTimeEmployee implements Payable {
    private String name;
    private double monthlySalary;

    public FullTimeEmployee(String name, double salary) {
        this.name = name;
        this.monthlySalary = salary;
    }

    @Override
    public double calculatePay() { return monthlySalary; }

    @Override
    public String getEmployeeInfo() { return name + " [Full-Time]"; }
}

class PartTimeEmployee implements Payable {
    private String name;
    private double hourlyRate;
    private int hoursWorked;

    public PartTimeEmployee(String name, double rate, int hours) {
        this.name = name;
        this.hourlyRate = rate;
        this.hoursWorked = hours;
    }

    @Override
    public double calculatePay() { return hourlyRate * hoursWorked; }

    @Override
    public String getEmployeeInfo() { return name + " [Part-Time]"; }
}

public class PayableDemo {
    public static void main(String[] args) {
        Payable[] employees = {
            new FullTimeEmployee("Alice", 45000),
            new PartTimeEmployee("Bob", 250, 80)
        };
        for (Payable emp : employees)
            System.out.printf("%s — Pay: P%.2f%n",
                emp.getEmployeeInfo(), emp.calculatePay());
    }
}
Output
Alice [Full-Time] — Pay: P45000.00 Bob [Part-Time] — Pay: P20000.00

The Payable interface defines a contract. Both employee types implement it differently, but are processed uniformly through the interface reference — interface-based polymorphism.

Seatwork 4
Abstract Class Hierarchies
▾

1. When would you choose an abstract class over an interface?

Answer

Choose an abstract class when related classes share common state (fields) and behavior (concrete methods), but need to enforce certain methods. Use an interface when unrelated classes share a capability.

abstract class Vehicle with shared fields (speed, fuel) and methods (startEngine()) — all vehicles share these. But interface GPS can be implemented by vehicles, phones, and watches — unrelated classes sharing a capability.

Activity 4
Banking System
▾
BankingSystem.java 
abstract class BankAccount {
    protected String owner;
    protected double balance;
    protected String accountNumber;

    public BankAccount(String owner, String accNum, double initial) {
        this.owner = owner;
        this.accountNumber = accNum;
        this.balance = initial;
    }

    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.printf("  + Deposited P%.2f%n", amount);
        }
    }

    public abstract void withdraw(double amount);
    public abstract double calculateInterest();

    public void displayInfo() {
        System.out.println("Account: " + accountNumber);
        System.out.println("Owner:   " + owner);
        System.out.printf("Balance: P%.2f%n", balance);
        System.out.printf("Interest: P%.2f%n", calculateInterest());
    }
}

class SavingsAccount extends BankAccount {
    private static final double RATE = 0.02;

    public SavingsAccount(String owner, String accNum, double initial) {
        super(owner, accNum, initial);
    }

    @Override
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.printf("  - Withdrawn P%.2f%n", amount);
        } else {
            System.out.println("  x Insufficient funds.");
        }
    }

    @Override
    public double calculateInterest() { return balance * RATE; }
}

class CheckingAccount extends BankAccount {
    private static final double RATE = 0.005;
    private static final double OVERDRAFT = 5000;

    public CheckingAccount(String owner, String accNum, double initial) {
        super(owner, accNum, initial);
    }

    @Override
    public void withdraw(double amount) {
        if (amount > 0 && (balance - amount) >= -OVERDRAFT) {
            balance -= amount;
            System.out.printf("  - Withdrawn P%.2f%n", amount);
        } else {
            System.out.println("  x Exceeds overdraft limit.");
        }
    }

    @Override
    public double calculateInterest() {
        return balance > 0 ? balance * RATE : 0;
    }
}

public class BankingSystem {
    public static void main(String[] args) {
        BankAccount[] accounts = {
            new SavingsAccount("Jayvee", "SA-001", 20000),
            new CheckingAccount("Maria", "CA-001", 15000)
        };

        for (BankAccount acc : accounts) {
            System.out.println("\n--- Account Details ---");
            acc.deposit(5000);
            acc.withdraw(3000);
            acc.displayInfo();
        }
    }
}
Output
--- Account Details --- + Deposited P5000.00 - Withdrawn P3000.00 Account: SA-001 Owner: Jayvee Balance: P22000.00 Interest: P440.00 --- Account Details --- + Deposited P5000.00 - Withdrawn P3000.00 Account: CA-001 Owner: Maria Balance: P17000.00 Interest: P85.00
Logic
All four OOP pillars working together:
  • Abstraction — BankAccount is abstract; withdraw() and calculateInterest() are abstract.
  • Encapsulation — Fields are protected. Rules enforced within methods.
  • Inheritance — Both accounts extend BankAccount, reusing deposit() and displayInfo().
  • Polymorphism — BankAccount[] array processes both types uniformly at runtime.

Library Management System
▾
LibrarySystem.java 
import java.util.ArrayList;

interface Borrowable {
    void borrowItem();
    void returnItem();
    boolean isAvailable();
}

class Book implements Borrowable {
    private String title, author;
    private boolean available;

    public Book(String title, String author) {
        this.title = title;
        this.author = author;
        this.available = true;
    }

    @Override
    public void borrowItem() {
        if (available) {
            available = false;
            System.out.println("  Borrowed: " + title);
        } else {
            System.out.println("  '" + title + "' is not available.");
        }
    }

    @Override
    public void returnItem() {
        available = true;
        System.out.println("  Returned: " + title);
    }

    @Override
    public boolean isAvailable() { return available; }

    @Override
    public String toString() {
        return String.format("%-28s by %-18s [%s]",
            title, author, available ? "Available" : "Borrowed");
    }
}

class Library {
    private ArrayList<Book> catalog = new ArrayList<>();

    public void addBook(Book b) { catalog.add(b); }

    public void showCatalog() {
        System.out.println("\n=== Library Catalog ===");
        for (Book b : catalog)
            System.out.println("  " + b);
    }
}

public class LibrarySystem {
    public static void main(String[] args) {
        Library lib = new Library();
        Book b1 = new Book("Clean Code", "Robert C. Martin");
        Book b2 = new Book("Head First Java", "Kathy Sierra");
        Book b3 = new Book("Effective Java", "Joshua Bloch");

        lib.addBook(b1); lib.addBook(b2); lib.addBook(b3);
        lib.showCatalog();

        b1.borrowItem();
        b2.borrowItem();
        b1.borrowItem(); // should fail
        lib.showCatalog();

        b1.returnItem();
        lib.showCatalog();
    }
}
Output
=== Library Catalog === Clean Code by Robert C. Martin [Available] Head First Java by Kathy Sierra [Available] Effective Java by Joshua Bloch [Available] Borrowed: Clean Code Borrowed: Head First Java 'Clean Code' is not available. === Library Catalog === Clean Code by Robert C. Martin [Borrowed] Head First Java by Kathy Sierra [Borrowed] Effective Java by Joshua Bloch [Available] Returned: Clean Code === Library Catalog === Clean Code by Robert C. Martin [Available] Head First Java by Kathy Sierra [Borrowed] Effective Java by Joshua Bloch [Available]
Key Learnings
  • Interface as contract — Borrowable defines capabilities without implementation details.
  • State management — available flag enforces business rules.
  • ArrayList over arrays — Dynamic collection management.
  • toString() override — Clean, formatted object output.

1. Design a class hierarchy for a vehicle rental system with an interface, abstract class, and two concrete classes.

VehicleRental.java 
interface Rentable {
    double calculateRentalCost(int days);
    String getRentalTerms();
}

abstract class Vehicle implements Rentable {
    protected String make, model;
    protected int year;
    protected double dailyRate;

    public Vehicle(String make, String model, int year, double rate) {
        this.make = make;
        this.model = model;
        this.year = year;
        this.dailyRate = rate;
    }

    public void displayInfo() {
        System.out.printf("%d %s %s — P%.0f/day%n", year, make, model, dailyRate);
    }
}

class Car extends Vehicle {
    public Car(String make, String model, int year, double rate) {
        super(make, model, year, rate);
    }

    @Override
    public double calculateRentalCost(int days) {
        return days >= 7 ? dailyRate * days * 0.9 : dailyRate * days;
    }

    @Override
    public String getRentalTerms() {
        return "Standard rental. 10% discount for 7+ days.";
    }
}

class SUV extends Vehicle {
    private static final double INSURANCE = 500;

    public SUV(String make, String model, int year, double rate) {
        super(make, model, year, rate);
    }

    @Override
    public double calculateRentalCost(int days) {
        return (dailyRate * days) + INSURANCE;
    }

    @Override
    public String getRentalTerms() {
        return "Premium rental. Includes P500 insurance fee.";
    }
}

public class VehicleRental {
    public static void main(String[] args) {
        Vehicle[] fleet = {
            new Car("Toyota", "Vios", 2024, 1500),
            new SUV("Ford", "Everest", 2024, 3500)
        };

        for (Vehicle v : fleet) {
            System.out.println("\n--- Rental Quote ---");
            v.displayInfo();
            System.out.println("Terms: " + v.getRentalTerms());
            System.out.printf("Cost for 10 days: P%.2f%n",
                v.calculateRentalCost(10));
        }
    }
}
Output
--- Rental Quote --- 2024 Toyota Vios — P1500/day Terms: Standard rental. 10% discount for 7+ days. Cost for 10 days: P13500.00 --- Rental Quote --- 2024 Ford Everest — P3500/day Terms: Premium rental. Includes P500 insurance fee. Cost for 10 days: P35500.00
